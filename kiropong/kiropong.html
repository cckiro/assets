The Problem

AI agents are powerful. But without guardrails, one bad prompt can delete your codebase.
  • Agents have unlimited tool access by default
  • No way to restrict capabilities per task or per agent
  • No delegation — one agent does everything
  • No audit trail — no warnings, no blocks, no logs
  • No way to share agent config across teams

In regulated environments — finance, healthcare, government — this is a non-starter.

Hooks — Runtime Guardrails

HookWhenCommandExit
agentSpawnAgent startsecho '🎯 Orchestrator active'0
preToolUse writeBefore writeecho '⚠️ Consider delegating'0
preToolUse shellBefore shellecho '⚠️ Consider delegating'0
stopAgent finishesecho '✅ Task complete'0

exit 0 — Allow

Action proceeds. Warning logged. Agent may self-correct.

exit 2 — Block

Action stopped. Agent must find another way — like delegating.

Matchers target specific tools. Without a matcher, the hook fires for every tool call.

Steering — The Brain

# Orchestrator Rules

## When to delegate
- Write tasks → write-agent
- Investigation → readonly
- Game building → game-builder

## When to act directly
- Simple reads, quick answers
- MCP documentation lookups

## Always
- Explain what you're doing
- Report back after delegation

Why steering matters

  • Rules in plain English
  • Loaded via file:// reference
  • Version controlled in git
  • Change rules → behavior changes immediately
  • No code changes needed
Your compliance team can read and approve steering rules. Try doing that with a neural network.

Skills — Swappable Task Specs

The game-builder agent is generic. Its prompt says: "follow the skill instructions exactly."

🏓

kiropong/SKILL.md

600×300 canvas · paddle physics · vs AI / 2 Player · First to 5 / Timed 45s

👾

space-invaders/SKILL.md

Same agent → different game

How skills work

  • Progressive loading — metadata at startup, full content on demand
  • Keyword triggers — "compare", "vs" activates safe-research
  • Reference files — supporting docs the agent reads during workflow

The pattern

Same agent + different skill = different output.
Swap the skill, don't rebuild the agent.
